使用中文简体回复和创建文档

## 技术栈

| 维度     | 技术选型                                                |
| -------- | ------------------------------------------------------- |
| 前端框架 | Vue 3.5 + TypeScript 5.9（严格模式）                    |
| 构建工具 | Vite 8 + vite-ssg（SSG 静态站点生成）                   |
| 路由     | vue-router 5（文件系统自动路由）                        |
| 样式     | Tailwind CSS v4 + shadcn-vue (new-york 风格, zinc 色调) |
| UI 组件  | reka-ui + lucide-vue-next 图标                          |
| 国际化   | vue-i18n v11（简体中文、繁体中文、英文、日文）          |
| 状态管理 | Pinia（favorites、navigation）                          |
| 代码规范 | ESLint 10 + oxlint + oxfmt + prettier                   |
| 后端服务 | Cloudflare Workers + Hono 框架                          |
| 部署平台 | Cloudflare Pages + Workers                              |

## 关键配置文件

- [vite.config.ts](mdc:vite.config.ts) — Vite 构建配置，插件链、路径别名、开发服务器（端口 5678）
- [components.json](mdc:components.json) — shadcn-vue 配置（new-york 风格，zinc 基础色，CSS 变量主题）
- [tailwind.config.js](mdc:tailwind.config.js) — Tailwind 扫描路径配置
- [tsconfig.app.json](mdc:tsconfig.app.json) — TypeScript 严格模式配置
- [src/main.ts](mdc:src/main.ts) — 应用入口（ViteSSG 模式）

## 核心规范

1. **包管理器**：使用 `pnpm`（版本 10.30.0）
2. **UI 库**：Shadcn Vue（new-york 风格），组件文档在 `docs/shadcn-components/`
3. **国际化**：所有用户可见字符串必须使用 `vue-i18n`，不得硬编码
4. **语言**：使用简体中文回复和创建文档
5. **路径别名**：`@` 指向 `src/`
6. **组件自动注册**：`src/components/` 下的组件无需手动 import
7. **自动导入**：`vue`、`vue-router` 的 Composable API 无需手动 import

## 新增工具页面流程

1. 在 `src/pages/` 下新建 `工具名.vue`（自动注册路由）
2. 在 `src/config/tools.ts` 中注册工具信息
3. 在 `src/i18n/zh-CN.ts`、`en.ts`、`zh-TW.ts`、`ja.ts` 中添加 i18n 翻译
4. 如需后端代理，在 `src/worker/routes/` 中添加 Hono 路由

## 代码规范要求

- TypeScript 严格模式：`strict`、`noUnusedLocals`、`noUnusedParameters` 均已开启
- 禁止硬编码字符串，必须使用 i18n
- 使用 `cn()` 工具函数（来自 `@/lib/utils`）合并 Tailwind 类名

## 编辑完成后置操作

1. **Git Commit 推荐**：每次编辑完成后，助手需主动推荐 Git Commit 信息（遵循 Conventional Commits 规范）
2. **代码检查**：运行 `pnpm lint-check` 进行 Lint 检查（oxlint + eslint）
3. **类型检查**：运行 `pnpm type-check` 进行 TypeScript 类型检查
